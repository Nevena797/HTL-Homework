HTTP (Hypertext Transfer Protocol)

the transfer of data over the internet - Postman

Introduction to HTTP

  • the delivery system for information on the internet
  • the foundation of the World Wide Web
  • way for web browsers and servers to talk to each other

HTTP Request - Response Cycle

a concept, web browser, a mobile app

  1. client -> an HTTP request message
  2. server receives, processes it, generate a response message
  3. the server -> response message back to the client
  4. client receives the response message, processes it
  5. client may repeating the cycle as needed

HTTP Request Methods

  • HEAD - asks the server for status
  • GET - asks the server to retrieve a resource
  • POST - asks the server to create a new resource
  • PUT - asks the server to edit/update an existing resource
  • DELETE - asks the server to delete a resource

HTTP Request Headers - name/value format

  • GET /api/data HTTP/1.1
  • Host: example.com -> specifies the hostname of the server
  • User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/89.0.4389.82 Safari/537.36 -> the client that is making the request
  • Accept: application/json -> MIME types of data that the client is willing to accept
  • Accept-Language: en-US,en;q=0. -> the preferred language(s) for the response
  • Authorization: Token abc123 -> access token for authentication
  • Cache-Control: no-cache -> caching directives for both requests and responses
  • Connection: keep-alive -> options for the connection handling between client and server
  • Referer: https://www.google.com/ - > specifies the URL of the page that linked to the current page
  • Pragma: no-cache -> specifies implementation-specific directives that might apply to any agent along the request-response chain
  • Content-Type -> This header specifies the MIME type of the data that is being sent in the body of the request, but it is not used in this example because this is a GET request without a request body.

HTTP Request Body

  • POST /v1/images/upload HTTP/1.1
  • Host: api.thecatapi.com -> the hostname of the server
  • x-api-key: API_KEY -> API key for authentication
  • Content-Length: 232 -> the length of the request body in bytes
  • Content-Type: image/jpeg -> uploading an image file named
  1. the server receives this request
  2. parse the request body
  3. use it to create a new entry in the Cat API database
  4. the server will then return a response
  5. image URL and the database ID

the message that a server sends back to a client

  7. 404 -> not found on the server
  8. 500 -> Internal Server Error
  9. 301 -> resource has been permanently moved to a new URL
