Beispielfragen – Grundlegende Webtechniken

Allgemeines über das World Wide Web

Was sind die wichtigsten Technologien, die das World Wide Web so erfolgreich gemacht haben?

  • Hypertext-Übertragungsprotokoll (HTTP)
  • Einheitlicher Ressourcen-Locator (URL)
  • Hypertext-Markup-Sprache (HTML)
  • Webbrowser
  • Domain Name System (DNS)
  • Suchmaschinen
  • JavaScript
  • Cascading Style Sheets (CSS)
  • Webserver und Datenbanken
  • Secure Sockets Layer (SSL) und HTTPS
  • Content-Delivery-Netzwerke (CDNs)
  • Responsive Design und mobile Webtechnologien
  • APIs und Webservices (REST, SOAP)
  • Soziale Medien und nutzergenerierte Inhalte

Wie werden Ressourcen (HTML-Seiten, Bilder usw.) im Web adressiert?

  • mithilfe von URLs (Uniform Resource Locators) adressiert
  • URLs bieten den Speicherort jeder Ressource

Was sind die allgemeinen Bestandteile einer URL?

  1. Protokoll: Gibt die Kommunikationsmethode an, normalerweise http oder https.
  2. Domain: Die Adresse des Servers (z. B. example.com).
  3. Port (optional): Der Port des Servers, standardmäßig 80 für http und 443 für https (z. B. :8080).
  4. Pfad: Der spezifische Speicherort der Ressource auf dem Server (z. B. /images/photo.jpg).
  5. Abfrage (optional): Schlüssel-Wert-Paare für dynamische Inhalte, beginnend mit ? (z. B. ?search=web).
  6. Fragment (optional): Ein Abschnitt innerhalb der Ressource, beginnend mit # (z. B. #section1).

Webserver

Was ist ein Webserver?

  • Computersystem oder eine Software
  • Websites hostet
  • Webinhalte über das Internet an Nutzer liefert
  • empfängt Anfragen von Clients (wie Browsern)
  • verarbeitet sie und liefert die angeforderten Ressourcen über die HTTP- oder HTTPS-Protokolle
  • Apache, Nginx und Microsoft IIS

Was macht ein Webserver?

  • speichert und liefert Websites
  • Nutzer ruft eine Website auf
  • sendet an den Browser des Nutzers

Wie können Inhalte auf einem Webserver gespeichert werden?

  • Statische Dateien: HTML, CSS, JavaScript, Bilder, Videos, Dokumente
  • Datenbanken:
    • Dynamische Inhalte: Benutzerdaten, Produktinformationen und Beiträge
    • Der Webserver ruft diese Daten basierend auf Benutzeranfragen ab
    • generiert dynamisch Seiten
  • Content-Management-Systeme (CMS):
    • WordPress oder Drupal
    • speichern Inhalte in einer Datenbank
    • Veröffentlichung von Inhalten - benutzerfreundliche Oberfläche
  • Cloud-Speicher:
    • insbesondere große Dateien wie Medien
    • AWS S3
    • über URLs abgerufen werden
    • die Hauptlast des Webservers reduzieren

Wie wird Inhalt auf Webseiten beschrieben?

  • HTML (HyperText Markup Language)
    • HTML strukturiert den Inhalt
    • definiert das Layout sowie die Bedeutung der Informationen
  • Elemente: Überschriften, Absätze, Bilder, Links, Listen
  • CSS (Cascading Style Sheets)
    • HTML-Elemente gestalten
    • formatieren
  • JavaScript
    • Interaktivität
    • dynamisches Verhalten

Webclient

Was ist ein Webclient?

  • Software: Programm, das Webinhalte anzeigt
  • Kommunikationspartner: Stellt Anfragen an Webserver und empfängt Antworten
  • Schnittstelle für Nutzer: Ermöglicht das Anzeigen und Interagieren mit Websites
  • Unterstützt Protokolle: Nutzt Protokolle wie HTTP oder HTTPS zur Datenübertragung

Was macht ein Webclient?

  • Anfragen senden: Fordert Inhalte von einem Webserver an (z. B. Websites).
  • Darstellen: Zeigt die Website für den Nutzer an
  • Interaktionen verarbeiten: Reagiert auf Klicks, Eingaben und andere Benutzeraktionen
  • Kommunikation verwalten: Tauscht Daten mit Servern für dynamische Inhalte

Sind Webclient und Webbrowser dasselbe?

Sie sind nicht genau dasselbe:

  1. Webclient:
    • kann ein Webbrowser sein
    • ist jede Software oder jedes Gerät, das mit einem Webserver interagiert
    • andere Anwendungen (z. B. mobile Apps oder Kommandozeilentools)
    • Beispiele: Discord, Teams
  2. Webbrowser:
    • eine spezifische Art von Webclient
    • Beispiele: Chrome, Firefox, Safari, Edge
    • Webinhalte anfordern, abrufen und darstellen

Hallo Welt

Was ist das HTML-Head-Element?

  • Das <head>-Element in HTML
  • enthält Meta-Informationen über die Webseite
  • Beispiele: Titel, Zeichenkodierung, Links zu Stylesheets und Skripten
  • zeigt keinen Inhalt auf der Seite an, liefert aber wichtige Daten für den Browser und Suchmaschinen

Was bedeutet <!DOCTYPE html>?

  • eine Deklaration des Dokumenttyps
  • hilft dem Browser, die Seite korrekt darzustellen
  • sicherstellt, dass er den Regeln und der Syntax von HTML folgt

Was sind die beiden obersten Elemente innerhalb des HTML-Elements?

Die beiden obersten Elemente innerhalb des <html>-Elements sind:

  • <head> – Enthält Metadaten wie Titel, Skripte, Stylesheets.
  • <body> – Enthält den sichtbaren Inhalt der Webseite.

Wie betont man Text und wie wird er im Browser angezeigt?

  • Das <em>-Tag
  • stellt den Text in den meisten Browsern standardmäßig kursiv dar

Wie markiert man wichtigen Text und wie wird er im Browser angezeigt?

  • Das <strong>
  • zeigt den Text standardmäßig fett an
