Basic Web Techniques

Understanding the Structure of a URL

Scheme, Server and Page

  • Scheme: the protocol used to access a resource on the internet http, https.
  • URL (Uniform Resource Locator): the address used to access a resource on a web server. It contains the scheme, domain name, path, and sometimes a query or fragment.
  • Server: a computer or system that hosts a website or web application, serving the files or content to users over the internet.
  • Page: is a specific file or document on the website, often written in HTML, that a user accesses via a URL. The page is part of the web content hosted on the server.

Web Browsers & Web Servers

  • Web Browser: software that allows users to access, retrieve, and view content on the internet, such as web pages, images, and videos, by interpreting HTML and other web technologies.
  • Web Server: a computer system that hosts websites and delivers web content to users' browsers over the internet when they request it, through the HTTP or HTTPS protocols.

Syntax

Every HTTP URL conforms to the syntax of a generic URI.

  • URI = scheme ":" ["//" authority] path ["?" query] ["#" fragment]
  • scheme: case-insensitive, http, https..
  • authority - an optional userinfo, a host subcomponent, (including but not limited to a hostname) or an IP address
  • An optional port
  • A path component
  • By convention, in http and https URIs, the last part of a path is named path info and it is optional
  • An optional query - query string of non-hierarchical data
  • An optional fragment component preceded by a hash (#)
