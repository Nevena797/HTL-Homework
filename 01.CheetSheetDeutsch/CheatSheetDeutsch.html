Verständnis des Aufbaus einer URL

Schema, Server und Seite

  • Schema: Das Protokoll, das verwendet wird, um eine Ressource im Internet zuzugreifen, z.B. http, https.
  • URL (Uniform Resource Locator): Die Adresse, die verwendet wird, um eine Ressource auf einem Webserver zuzugreifen. Sie enthält das Schema, den Domainnamen, den Pfad und manchmal eine Abfrage oder ein Fragment.
  • Server: Ein Computer oder System, das eine Website oder Webanwendung hostet und die Dateien oder Inhalte den Nutzern über das Internet bereitstellt.
  • Seite: Eine spezifische Datei oder ein Dokument auf der Website, das oft in HTML geschrieben ist und das ein Nutzer über eine URL aufruft. Die Seite ist Teil des Webinhalts, der auf dem Server gehostet wird.
  • Webbrowser: Software, die es Nutzern ermöglicht, auf Inhalte im Internet zuzugreifen, diese abzurufen und anzuzeigen, wie zum Beispiel Webseiten, Bilder und Videos, index sie HTML und andere Webtechnologien interpretiert.
  • Webserver: Ein Computersystem, das Websites hostet und Webinhalte an die Browser der Nutzer über das Internet liefert, wenn diese sie anfordern, über die HTTP- oder HTTPS-Protokolle.

Syntax

Jede HTTP-URL entspricht der Syntax eines generischen URI.

  • URI = Schema ":" ["//" Autorität] Pfad ["?" Abfrage] ["#" Fragment]
  • Schema: nicht groß-/kleinschreibungsempfindlich, z.B. http, https...
  • Autorität - eine optionale Benutzerinfo, eine Host-Komponente (einschließlich, aber nicht beschränkt auf einen Hostnamen) oder eine IP-Adresse.
  • Ein optionaler Port
  • Eine Pfad-Komponente
  • Nach Konvention wird in http- und https-URIs der letzte Teil eines Pfads als pathinfo bezeichnet und ist optional.
  • Eine optionale Abfrage - Abfragezeichenfolge für nicht-hierarchische Daten
  • Eine optionale Fragmentkomponente, die durch ein Hash-Zeichen (#) eingeleitet wird

Webserver

Ein Webserver ist eine Computersoftware und die zugrunde liegende Hardware.

  • Akzeptiert Anfragen über HTTP/HTTPS
  • Der Server antwortet mit Inhalten oder einer Fehlermeldung
  • Speichert Ressourcen
  • Am unteren Ende: Eingebettete Systeme, wie z.B. ein Router, der einen kleinen Webserver als Konfigurationsschnittstelle betreibt
  • Eine Ressource, die von einem Webserver gesendet wird, kann eine Datei sein (statischer Inhalt) oder dynamischer Inhalt
